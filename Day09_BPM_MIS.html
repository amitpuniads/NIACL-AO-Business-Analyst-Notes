What is MIS?

Definition: MIS is a computer-based system that provides managers with tools to organize, evaluate, and efficiently manage information for strategic decision-making.

üîß Part 3: Software Development Life Cycle (SDLC)

What is SDLC?

Definition: SDLC is a structured process for developing software systems, from initial planning to deployment and maintenance.

SDLC Phases (6 Main Phases)

Phase 1: Planning & Requirement Analysis
‚Ä¢ Identify project scope and objectives
‚Ä¢ Gather business requirements from stakeholders
‚Ä¢ Conduct feasibility study (technical, operational, economic)
‚Ä¢ Define project timeline and resources
Deliverable: Requirement Specification Document
Phase 2: System Design
‚Ä¢ Create high-level design (architecture)
‚Ä¢ Design database schema, UI/UX mockups
‚Ä¢ Define technical specifications
‚Ä¢ Plan security and data flow
Deliverable: Design Document Specification (DDS)
Phase 3: Implementation/Development
‚Ä¢ Write code based on design documents
‚Ä¢ Follow coding standards and best practices
‚Ä¢ Conduct unit testing during development
‚Ä¢ Use version control (Git)
Deliverable: Working Software Code
Phase 4: Testing
‚Ä¢ Unit Testing (individual components)
‚Ä¢ Integration Testing (combined modules)
‚Ä¢ System Testing (complete application)
‚Ä¢ User Acceptance Testing (UAT by business users)
Deliverable: Test Reports, Bug Fixes
Phase 5: Deployment
‚Ä¢ Release software to production environment
‚Ä¢ Conduct user training sessions
‚Ä¢ Provide documentation and manuals
‚Ä¢ Monitor initial performance
Deliverable: Live Application
Phase 6: Maintenance
‚Ä¢ Fix bugs reported by users
‚Ä¢ Implement minor enhancements
‚Ä¢ Provide ongoing technical support
‚Ä¢ Plan for major upgrades
Deliverable: Updated Software Versions

Popular SDLC Models

Model Approach Best For Limitation
Waterfall Sequential, linear phases Clear, fixed requirements No flexibility for changes
Agile Iterative, incremental Evolving requirements Requires active user involvement
V-Model Testing at each phase High reliability systems Rigid, expensive
Spiral Risk-driven iterations Large, complex projects Complex to manage
üìã Example: NIACL Policy Management System Development

Planning: Identify need for digital policy issuance system, gather requirements from agents and underwriters

Design: Create database for policies, design web interface for policy creation

Development: Code policy creation module, payment gateway integration

Testing: Test with sample policies, conduct UAT with branch managers

Deployment: Roll out to all branches, train staff on new system

Maintenance: Fix bugs, add new policy types as required

üèÉ Part 4: Agile & Scrum Framework

What is Agile?

Definition: Agile is an iterative approach to software development that focuses on continuous delivery, collaboration, and flexibility to change.
Hindi: ‡§è‡§ú‡§æ‡§á‡§≤ - ‡§è‡§ï ‡§™‡•Å‡§®‡§∞‡§æ‡§µ‡•É‡§§‡•ç‡§§ ‡§¶‡•É‡§∑‡•ç‡§ü‡§ø‡§ï‡•ã‡§£ ‡§ú‡•ã ‡§®‡§ø‡§∞‡§Ç‡§§‡§∞ ‡§µ‡§ø‡§§‡§∞‡§£, ‡§∏‡§π‡§Ø‡•ã‡§ó ‡§î‡§∞ ‡§™‡§∞‡§ø‡§µ‡§∞‡•ç‡§§‡§® ‡§ï‡•á ‡§≤‡§ø‡§è ‡§≤‡§ö‡•Ä‡§≤‡•á‡§™‡§® ‡§™‡§∞ ‡§ï‡•á‡§Ç‡§¶‡•ç‡§∞‡§ø‡§§ ‡§π‡•à‡•§

Agile Core Values (4 Values from Agile Manifesto)

1. Individuals and interactions > Processes and tools
2. Working software > Comprehensive documentation
3. Customer collaboration > Contract negotiation
4. Responding to change > Following a plan

Scrum Framework

Definition: Scrum is the most popular Agile framework that organizes work into fixed-length iterations called "Sprints" (usually 2-4 weeks).

Scrum Roles (3 Key Roles)

Role Responsibility Insurance Example
Product Owner Defines product vision, prioritizes backlog, represents stakeholders Insurance business manager defining features for claim system
Scrum Master Facilitates Scrum process, removes blockers, coaches team Agile coach ensuring team follows Scrum practices
Development Team Self-organizing team that builds the product (5-9 members) Developers, testers, BA working on claim portal

Scrum Artifacts (3 Main Artifacts)

1. Product Backlog: Prioritized list of features/requirements for entire product
Example: List of features for NIACL mobile app (login, view policies, file claim, chat support)
2. Sprint Backlog: Subset of product backlog selected for current sprint
Example: In 2-week sprint, team commits to complete "login" and "view policies" features
3. Product Increment: Sum of all completed product backlog items, must be in usable condition
Example: After Sprint 1, working login and policy viewing functionality ready for demo

1. Sprint Planning (Sprint ‡§ï‡•Ä ‡§Ø‡•ã‡§ú‡§®‡§æ): Team selects work for upcoming sprint
Duration: 2-4 hours for 2-week sprint
2. Daily Standup/Daily Scrum (‡§¶‡•à‡§®‡§ø‡§ï ‡§¨‡•à‡§†‡§ï): 15-minute daily sync
Team answers: What did I do yesterday? What will I do today? Any blockers?
3. Sprint Review (Sprint ‡§∏‡§Æ‡•Ä‡§ï‡•ç‡§∑‡§æ): Demo completed work to stakeholders
Duration: 1-2 hours, gather feedback
4. Sprint Retrospective (Sprint ‡§™‡•Ç‡§∞‡•ç‡§µ‡§æ‡§µ‡§≤‡•ã‡§ï‡§®): Team reflects on process improvements
Duration: 1 hour, discuss what went well, what didn't, action items
5. Backlog Refinement (‡§¨‡•à‡§ï‡§≤‡•â‡§ó ‡§∂‡•ã‡§ß‡§®): Ongoing activity to clarify and estimate backlog items
Duration: ~10% of sprint time

Agile vs Waterfall Comparison

Aspect Waterfall Agile
Approach Sequential, linear Iterative, incremental
Requirements Fixed at start Evolve throughout project
Testing After development phase Continuous, every sprint
Customer Involvement Beginning and end Throughout the project
Delivery One final delivery Working software every sprint

Tip: In insurance, Agile is best for customer-facing applications (mobile apps, portals) while Waterfall may suit compliance-driven projects with fixed regulations.

Set 1: BPM Fundamentals (Q1-Q10)

Q1. Which phase of BPM lifecycle involves creating flowcharts and BPMN diagrams?
Solution: Model Phase (‡§Æ‡•â‡§°‡§≤ ‡§ö‡§∞‡§£) - This is where visual representations of the process are created.
Q2. In insurance claim settlement, which type of business process does it represent?
Solution: Core Process - It directly creates value for customers and is a primary business activity.
Q3. A process improvement reduced claim settlement time from 10 days to 7 days. What is the percentage improvement?
Solution: [(10-7)/10] √ó 100 = 30% improvement

Solution: Monitor Phase (‡§®‡§ø‡§ó‡§∞‡§æ‡§®‡•Ä ‡§ö‡§∞‡§£) - This phase tracks performance metrics and identifies issues.
Q5. If a process has 8 steps and optimization reduces it to 5 steps, what is the reduction percentage?
Solution: [(8-5)/8] √ó 100 = 37.5% reduction in steps
Q6. HR department providing recruitment services to operations team is an example of which process type?
Solution: Support Process - HR enables core processes but doesn't directly create customer value.
Q7. In BPM, what is the final phase that focuses on continuous improvement?
Solution: Optimize Phase (‡§Ö‡§®‡•Å‡§ï‡•Ç‡§≤‡§® ‡§ö‡§∞‡§£) - Analyzes data and implements improvements.
Q8. A company processes 500 claims/month. After BPM implementation, it processes 650 claims. What is the efficiency gain?
Solution: [(650-500)/500] √ó 100 = 30% efficiency gain
Q9. Risk management and compliance audits represent which type of process?
Solution: Management Process - These govern and control overall operations.
Q10. What are the three main types of business processes?
Solution: Core Process, Support Process, and Management Process

Q11. Which MIS component includes servers, computers, and scanners?

Q14. How many main components are there in an MIS?
Solution: 5 components - Hardware, Software, Data, Procedures, and People
Q15. A manager requests a specific customer's complete claim history. This is which type of report?

Q16. What is the main difference between a Dashboard and a Report?

Q17. Branch manager analyzes state-level sales, then drills down to district and branch level. This is?
Solution: Drill-Down Report (‡§µ‡§ø‡§∏‡•ç‡§§‡•É‡§§ ‡§∞‡§ø‡§™‡•ã‡§∞‡•ç‡§ü) - Allows detailed analysis by drilling into summary data.

Solution: Software - Programs and applications used for data processing and analysis.
Q19. An MIS generates 50 reports daily. If 40 are scheduled and rest are exceptions, what percentage are exception reports?
Solution: Exception reports = 50 - 40 = 10
Percentage = (10/50) √ó 100 = 20%
Q20. What is the primary benefit of MIS in insurance companies?
Solution: Quick access to accurate data for better decision-making, improved efficiency, and enhanced customer service through automation and real-time insights.

Set 3: SDLC Phases (Q21-Q30)

Q21. In which SDLC phase is the feasibility study conducted?
Solution: Planning & Requirement Analysis - This is the first phase where feasibility (technical, operational, economic) is assessed.
Q22. What is the deliverable of the System Design phase?
Solution: Design Document Specification (DDS) - Contains architecture, database schema, UI mockups, and technical specifications.
Q23. Which SDLC phase includes User Acceptance Testing (UAT)?
Solution: Testing Phase - UAT is the final testing where business users validate the system meets requirements.
Q24. Bug fixing and minor enhancements after deployment occur in which phase?
Solution: Maintenance Phase - Ongoing support, bug fixes, and minor updates happen here.
Q25. How many main phases are there in a typical SDLC?
Solution: 6 phases - Planning, Design, Implementation, Testing, Deployment, and Maintenance
Q26. Which SDLC model follows a sequential, linear approach with no flexibility for changes?
Solution: Waterfall Model - Phases flow sequentially like a waterfall, difficult to go back and make changes.
Q27. A project took 2 months for planning, 3 months for design, 6 months for development, 2 months for testing, and 1 month for deployment. What is the total project duration?
Solution: Total = 2 + 3 + 6 + 2 + 1 = 14 months
Q28. Which SDLC model is best for projects with evolving requirements?
Solution: Agile Model - It's iterative and flexible, welcoming changes throughout development.
Q29. Writing code and conducting unit testing occurs in which phase?
Solution: Implementation/Development Phase - Developers write code and perform unit testing during development.
Q30. What are the four types of testing in the Testing phase?
Solution: 1. Unit Testing, 2. Integration Testing, 3. System Testing, 4. User Acceptance Testing (UAT)

Set 4: Agile & Scrum (Q31-Q40)

Q31. What is the typical duration of a Sprint in Scrum?
Solution: 2-4 weeks (most commonly 2 weeks). It's a fixed time-box that doesn't change mid-sprint.
Q32. Who is responsible for prioritizing the Product Backlog?
Solution: Product Owner - They define product vision and prioritize backlog items based on business value.
Q33. How long should a Daily Standup meeting last?
Solution: 15 minutes maximum - It's a quick daily sync, not a detailed discussion meeting.
Q34. Which Scrum ceremony focuses on demonstrating completed work to stakeholders?
Solution: Sprint Review - Team demos the working increment and gathers feedback.
Q35. How many core roles are there in Scrum?
Solution: 3 roles - Product Owner, Scrum Master, and Development Team
Q36. A team completes 50 story points in a 2-week sprint. What is their velocity?
Solution: Velocity = 50 story points per sprint (or 25 story points per week)
Q37. Which Agile value states "Working software over comprehensive documentation"?
Solution: The second value from the Agile Manifesto - While documentation is valuable, working software is valued more.
Q38. In Scrum, who removes blockers and facilitates the process?
Solution: Scrum Master - They serve the team by removing impediments and coaching Scrum practices.
Q39. A product backlog has 200 story points. If team velocity is 40 points/sprint, how many sprints to complete?
Solution: Number of sprints = 200 / 40 = 5 sprints
Q40. Which artifact represents the sum of all completed product backlog items?
Solution: Product Increment - Must be in usable, "done" condition at the end of each sprint.

Set 5: Scenario-Based Questions (Q41-Q50)

Q41. NIACL wants to build a mobile app for customers. The requirements are unclear and may change. Which SDLC model should they use?
Solution: Agile/Scrum - Best for evolving requirements, allows flexibility, and delivers working features incrementally with customer feedback.
Q42. A BPM team reduced process steps from 12 to 8 and time from 15 days to 10 days. What is the time reduction percentage?
Solution: [(15-10)/15] √ó 100 = 33.33% time reduction
Q43. In Daily Standup, team members answer three questions. What are they?
Solution: 1. What did I do yesterday? 2. What will I do today? 3. Are there any blockers/impediments?
Q44. A manager needs to see only those policies where premium is overdue by more than 60 days. Which MIS report type?
Solution: Exception Report - It shows only exceptional cases (premiums overdue > 60 days) that need attention.
Q45. Development team has 7 members. After Sprint Planning, how many should be on the team for that sprint?
Solution: 7 members - Development team size doesn't change during a sprint. They commit as a whole team.
Q46. A compliance audit system needs to follow strict regulations with clear, fixed requirements. Which model is best?
Solution: Waterfall Model - Best for well-defined, stable projects with fixed requirements and regulatory compliance needs.
Q47. NIACL processes 10,000 claims/month with 5% requiring manual review. How many are auto-processed?
Solution: Manual review = 10,000 √ó 0.05 = 500
Auto-processed = 10,000 - 500 = 9,500 claims
Q48. In a 2-week sprint, how much time should be allocated for Sprint Retrospective?
Solution: Approximately 1-1.5 hours for a 2-week sprint. It's typically 45 minutes per week of sprint.
Q49. Which BPM phase comes immediately after "Monitor"?
Solution: Optimize Phase - After monitoring performance and identifying issues, the next step is optimization and improvement.
Q50. A team planned 60 story points but completed only 45 in the sprint. What is their achievement percentage?
Solution: Achievement % = (45/60) √ó 100 = 75%

Bonus Questions (Q51-Q55)

Q51. What is the ideal Development Team size in Scrum?
Solution: 5-9 members - Small enough to be agile, large enough to complete significant work each sprint.
